AI 챗봇 제작 및 배포 가이드

AI Studio vs NotebookLM: 어떤 도구를 써야 할까요?

두 도구는 목적이 다릅니다. 이 프로젝트에는 두 가지 모두, 혹은 목적에 맞게 선택하여 사용할 수 있습니다.

  • Google AI Studio: 챗봇의 '두뇌'를 만들고 테스트하기에 최적입니다. 프롬프트를 설계하고, 챗봇의 말투, 역할, 응답 방식을 빠르게 프로토타이핑할 수 있습니다. '명상 스크립트 생성', '사용자 질문 유도 프롬프트' 등을 여기서 만들고 테스트하는 것이 효율적입니다. API 키를 발급받아 지금 보고 계신 이런 웹 앱에 연동할 수 있습니다.
  • NotebookLM: 제공된 자료(연구 논문, 보고서 등)에 기반하여 답변하는 '연구 보조 AI'입니다. "OOO 연구 논문에 따르면 직장인 고통의 원인은 무엇이야?"와 같은 질문에 정확한 답변을 얻는 데 특화되어 있습니다. 챗봇이 제공할 정보의 '지식 베이스'를 구축하고 사실 확인을 하는 데 사용하면 좋습니다.
  • 결론: AI Studio로 챗봇의 대화 능력을 만들고, NotebookLM으로 챗봇이 참고할 전문 지식 소스를 관리하는 하이브리드 방식이 가장 이상적입니다.

HTML/CSS/JS vs Python: 어떤 기술 스택이 나을까요?

결론부터 말하면, HTML/CSS/JS 방식이 압도적으로 유리합니다.

  • 접근성 및 배포: HTML/CSS/JS로 만든 웹 앱은 URL만 있으면 누구나, 어떤 기기(PC, 모바일, 태블릿)에서든 바로 사용할 수 있습니다. Python 실행 파일(.exe)은 사용자가 직접 다운로드하고 설치해야 하는 허들이 있으며, 특정 운영체제(예: Windows)에서만 작동합니다.
  • 상호작용 및 UI: 지금 보고 계신 화면처럼 동적인 차트, 부드러운 애니메이션, 세련된 UI를 구현하는 데 웹 기술이 훨씬 강력하고 효율적입니다.
  • 확장성: 웹 앱은 Google AI Studio 등 다양한 외부 서비스(API)와 연동하기가 매우 쉽습니다. Python으로 UI를 만들고 외부 서비스와 연동하는 것은 훨씬 복잡합니다.

따라서 사용자와의 상호작용이 중요한 챗봇 애플리케이션은 웹 기술(HTML, CSS, JavaScript)로 제작하는 것이 표준적인 접근 방식입니다.

HTML/CSS/JS 앱 배포 방법

완성된 HTML 파일을 전 세계인이 접속할 수 있도록 배포하는 간편한 무료 서비스들이 많습니다.

  1. GitHub Pages: 가장 대중적인 방법입니다.
    • 코드를 저장하는 공간인 '저장소(repository)'를 GitHub에 만듭니다.
    • 이 HTML 파일을 `index.html` 이라는 이름으로 저장소에 올립니다.
    • 저장소 설정에서 'Pages' 메뉴를 찾아 배포를 활성화하면, 몇 분 안에 `사용자이름.github.io/저장소이름` 형태의 주소가 생성됩니다.
  2. Netlify / Vercel: 좀 더 전문적이고 빠른 배포를 원할 때 사용합니다. GitHub 계정과 연동하여 저장소에 코드를 올리기만 하면 자동으로 배포되는 편리한 기능을 제공합니다.

이러한 서비스를 이용하면 서버를 직접 관리할 필요 없이 간편하게 웹 애플리케이션을 배포하고 운영할 수 있습니다.
